Vue的生命周期(1)

一、Vue生命周期

在Vue官方文档中有这样一张图，很简洁明了地介绍了Vue的生命周期钩子。
Vue生命周期
其中有这几个生命周期。
beforeCreate,created,beforeMount,Mounted,beforeUpdate,updated,beforeDestroy,destroyed

二、源码理解

为了便于理解生命周期，不再用错钩子函数，我看了相关的源码，希望深入理解。
Vue本身是构造函数，new Vue(options)时会进行以下操作：

1、首先调用其原型上的初始化_init()方法

其他不理解的代码可以先放下，重点在于callHook(vm, ‘beforeCreate’)和callHook(vm, ‘created’)。
可以看到在beforeCreate生命周期只是完成了一些事件的初始化，而调用created生命周期前完成了数据状态的初始化，还没有渲染DOM到文档！！因此还不能操作DOM。

2、created生命周期钩子之后的initState()

在initState()函数中会分别对props,methods,data,computed,watch进行初始化，分别对应： initProps()|initData()|initComputed|initWatch()
在这里，我通过initData()来了解Vue的数据响应。

这里的意思是将data中的key值经过proxy代理，可以在之后直接通过this.属性的形式访问;然后通过observe(data)来实现响应式，中心原理是通过Object.defineProperty()进行访问描述符get()|set()函数的改写。

对data对象的每个属性执行defineReactive():

(1)将数据的属性转为访问器属性，对属性进行递归调用observe()，保证嵌套的属性都可以被监听
？？(2)在get()时，添加订阅者；在set()时，通过dep.notify()通知watcher数据发生了改变。

3、created()钩子之后，beforeMount()钩子之前

created()钩子被调用之后，触发created,继续执行vm.$mount(vm.$options.el)。
（1）获取模板。进入$mount会先获取挂载el节点，判断是否传入render函数，如果没有则判断是否传入模板template，没有模板的话，会选取外层HTML（getOuterHTML(el)）作为模板。
（2）有了模板之后，会调用compileToFunctions()将将模板(template)字符串编译为渲染函数(render),并将渲染函数赋值给 vm.$options.render 选项。

4、beforeMount()之后，Mounted()之前

生成render函数之后，会进入mountComponent()函数来挂载组件，在组件实例对象上添加 $el 属性，调用beforeMount()钩子函数之后组件将开始挂载；
接着会执行vm._wacher = new Watcher(vm, updateComponent, noop)；
最后调用mounted()钩子，因此在mounted()之前就有了DOM。
