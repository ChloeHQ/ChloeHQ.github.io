二、进阶版

虽然以上两种方法都可以解决题目所述问题，但是它们都会改变原链表的值或者结构，有没有什么方法可以维持原有链表不变呢。我想到在《剑指offer》中其实遇到过一道题——判断链表中是否成环，书中所用方法是设置快慢两个指针，快指针每次前进一步，慢指针每次前进两步，如果成环那么它们一定可以相遇。
这个题目是否可以利用这一特点呢？受到牛客网上 求一个大大的offer 小姐姐启发，得出下面的数学推导：

x: 环前面的路程
c: 环的长度
a: 入口到相遇点的距离
假设有快慢两个指针： pFast,pSlow
快指针每次走两步，慢指针每次一步，这样当它们在B点相遇时：
此时慢指针走的路程为 Sslow = x + m c + a
快指针走的路程为 Sfast = x + n c + a
由于2S_slow = S_fast，有：
2(x+mc+a) = x+nc+a =>
x = (n-2m-1)c + c-a，意思就是 环前面的路程=多个环的长度+BC之间的距离，
因此 解决方法就是当两个指针相遇后，设置一个指针从Head开始遍历链表，另一指针从相遇点开始遍历链表，步长都为1，它们再次相遇时，正好会是环的入口节点C。
