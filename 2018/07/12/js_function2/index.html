不常用到的htmL元素方法（下）

最近看到一篇很有意思的文章，原文作者是David Gilbertson.
其中介绍了我们平时不常用的JS操作DOM的方法，看过后确实发现部分很方便却冷门，因此萌生了代码验证以及笔记记录的念头。

英文原文

7、insertAdjacentElement()

我之前从来没有使用过这个DOM操作方法（也可能是见过但是没用过…），现在了解到它的作用类似于appendChild()方法 ，但能够更好地控制插入子元素的具体位置。
parentEl.insertAdjacentElement(‘beforeend’, newEl) 与 parentEl.appendChild(newEl) 的作用是一样的，但除此之外，你还可以指定beforebegin,afterbegin,afterend这几个参数值，元素将按这些值的名称所示插入相应的位置。

8、contains()

在需要知道某个元素是否被包含在另一个元素中时，可以使用contains()方法，返回的是bool值。

9、dialog 元素的三大法宝

<dialog> 是一个相对较新的元素，是HTML5新增的，它带来了两个还算能用的方法show() 和 close() ，和一个非常棒的方法showModal()。
showModal()方法能够将 <dialog>元素显示在页面的顶层，居中对齐，这正是所期望的模态窗口行为。我们无需指定 z-index，或者手动添加一个灰色的背景，也不需要监听 esc 按键以关闭此窗口。浏览器能够理解模态窗口的工作方式，并自动完成你所期望的行为。
注意： 这个方法只有Chrome和Safari支持。

效果：点击按钮会使对话框显示在文档最上层最中间的位置。

10表单

我们知道，<form> 有一个 submit() 方法。但是没有注意到的是表单还有一个 reset() 方法，而且当需要对表单元素进行验证时，还可以调用 reportValidity() 方法。
此外，还可以通过对表单的 elements 属性加上元素的 name 特性 的方式调用它的属性。例如myFormEl.elements.email 将返回属于某个<form> 中的 < input name=”email” /> 元素（‘属于’并不代表它一定是一个‘子元素’）。
elements并不会返回一个元素列表，而是返回一个控件列表（显然它不是一个数组，因为没必要这么做）。
例如：

现在有三个input，类型为radio,每个都有相同的名称 ipt。 form1.elements.ipt 将返回一个单选按钮集集的引用（一个控件，三个元素）。
而 form1.elements.ipt.value 将返回所选中的（即checked）单选按钮的值。
!(form.elements.name)[http://oz572skeg.bkt.clouddn.com/form.png]
这种语法看起来非常古怪，让我们来分解一下看看：formEl 是一个元素，elements 则对应 HTMLFormControlsCollection接口，这并非一个真正的数组，其中的每一项内容也未必代表一个 HTML 元素。ipt是多个单选按钮的集合，只是因为他们具有相同的 name 特性才聚集在一起（RadioNodeList 接口就是为此而生的），而 value 则返回该集合中所选中的那个单选按钮的 value 特性。
