那些我遗漏的CSS特性(1)

一些前端CSS小TIPS总结（上篇）

1、position: sticky

设置 position: sticky会让元素在页面滚动时如同在正常流中，但当其滚动到相对于视口的某个特定位置时就会固定在屏幕上，如同fixed一般。这个属性值是一个较新的CSS属性，在浏览器兼容性上会差一些，但在不兼容的浏览器中会被忽略并会退到正常的滚动情况。

这样的话具有item2这个class的元素在页面滚动时像是在正常流中，但是当它距离视口10px时，就会固定在屏幕上，它后面的元素会从它下层滑过。

2、Flex弹性布局

弹性盒子（Flexbox）布局是一种一维布局方法，一维的意思是内容是按行或者列来布局。我们可以使用 display: flex来将元素变为弹性布局。该容器的直接子元素会变为弹性项（flex item），并按行排列。

2.1 弹性布局的轴

弹性布局的轴有主轴和交叉轴，默认的flex-direction值为row，row代表了文本的行文方向。
交叉轴是和主轴垂直的一条轴，如果flex-direction是row并且弹性项是按照行内方向排列的，那么交叉轴就是块级元素的排列方向；如果flex-direction是column那么弹性项就会以块级元素排列的方向排布，然后交叉轴就会变为row。

2.2 弹性项的方向和次序

当然，有很多属性可以决定弹性项的次序——row|row-reverse|column|column-reverse
也可以通过order来改变一个弹性项的次序(小的在前），但是有一点容易被遗忘的是：许多人喜欢用键盘操作tab键切换元素，而 tab的顺序是页面内元素在源码中的顺序而非显示顺序

3、grid layout 网格布局

CSS网格布局（grid layout）是一种二维布局技术。二维意味着按照行和列来排布内容。为容器元素设置 display: grid来实现网格布局，并且使用grid-template-columns和grid-template-rows属性来控制网格中的行与列。

3.1基于行、列的基本定位方法

定位网格元素最简单的方式是使用基于行/列的定位方法，只需告诉浏览器从哪一排到哪一排来进行合并。
例如，如果需要一个2*2的网格区域，可以将指定元素从第一行开始到第三行、从第一列开始到第三列，这样就可以覆盖到四个单元格。

可以用缩写来表示：grid-column和grid-row，其中第一个值代表起始值，第二个值代表结束值。

3.2基于命名区域的定位方法

可以通过 命名区域（named areas）来定位网格中的元素。如果用这种方式，需要通过设置 grid-area属性给每个元素一个名字，然后在grid容器中通过 grid-template-areas属性的值来描述布局方式。
