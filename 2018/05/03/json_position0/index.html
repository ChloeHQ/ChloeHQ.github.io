报错：Unexpected token ﻿in JSON at position 0

问题说明

最近在做的一个Vue项目，在从接口获取json格式数据时，用JSON.parse(res)居然会报错
Unexpected token ﻿ in JSON at position 0

但是用JSON.parse(res.substr(1))却可以执行成功
然而两个字符串乍一看却完全一样，这让我很疑惑。

原因探索

F12仔细查看返回结果，发现在JSON字符串前有一个编码为’\ufeff’的字符

BOM是用来判断文本文件是哪一种Unicode编码的标记，其本身是一个Unicode字符（”\uFEFF”），位于文本文件头部。
在不同的Unicode编码中，对应的bom的二进制字节如下：
|Bytes|Encoding|
|:———-:|:———–:|
|FE FF|UTF16BE|
|FF FE|UTF16LE|
|EF BB|BF UTF8|
所以我们可以根据文件头部的几个字节和上面的表格对应来判断该文件是哪种编码形式
将其截取掉就可以
