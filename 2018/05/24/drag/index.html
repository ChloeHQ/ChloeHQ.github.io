HTML5新特性：拖放

一、过程

1、设置元素为可拖放的

为想要拖拽的元素添加draggable特性，这样元素的dragstart事件才可以被触发。
（默认auto情况下img和带有href属性的元素是可以被拖拽的，其他不行）

2、被拖拽元素的生命周期

(1)dragstart:被拖拽元素开始被拖拽时候触发

  • event.dataTransfer的大部分设置在这里配置；
  • 如果调用event.preventDefault()，会阻止拖拽行为，导致后续拖拽行为不被触发；
  • 触发dragstart事件后，其他元素的mouseover,mousemove,mouseenter,mouseout,mouseleave事件都不会触发。

(2)drag：当被元素正在被拖拽时触发
(3)dragend:当拖拽行为结束后触发（例如释放鼠标）

3、目标元素生命周期

(1)dragenter :当被拖拽元素进入目标元素时触发
(2)dragover:当被拖拽元素在目标元素上移动时触发

  • dragover事件默认行为是不允许被拖拽元素在其他元素上释放或者放置的（可以理解为默认不允许触发drop事件），一般会event.preventDefault()来阻止默认行为以便能够触发drop事件。
    (3)drop当被拖拽元素在目标元素上，并且释放鼠标左键时触发
    (4)dragleave:被拖拽元素离开目标元素时触发
4、整体声明周期

dragstart -> drag -> dragenter -> dragover -> drop -> dragleave -> dragend

二、数据传递对象DataTransfer

DataTransfer对象用于在配置拖拽行为效果，在拖拽过程中的各事件间传递数据。
e.dataTransfer
(1)effectAllowed和dropEffect属性
用于配置在拖拽过程中鼠标指针的类型以便提示用户后续可进行的操作；
控制drop事件触发与否。
(2)方法

  • void addElement(ele): 添加一起跟随鼠标移动的元素，仅在dragstart事件中调用。（兼容性？
  • void setDragImage((Element)image,(long)x,(long)y)：设置拖动过程中跟随鼠标一起移动的图片。
  • boolean setData({DOMString}format, {DOMString} data):将制定格式的数据赋给DataTransfer.
    format值范围为URL|Text或各种MIME类型，其实Text会被自动映射为text/plain
    仅在dragstart事件中被调用
  • DOMString getData({DOMString} format)：从DataTransfer对象中获取指定格式的数据。
  • void clearData({DOMString} format):从DataTransfer对象中删除指定格式的数据（仅在dragstart事件中调用）
