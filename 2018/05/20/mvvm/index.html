手写一个MVVM框架（1）

一、理解MVVM

（1）MVC: 传统MVC是应用到后端的框架 用户操作视图层，触发一些事件，从而通过Controller修改Model，然后Model又可以反映到View上。

（2）MVVM:借鉴MVC，视图和模型是分离的

对应关系如下：
Model: data:{}
ViewModel: var vm = new Vue({ … }) 是View和Model连接的桥梁
View通过事件绑定操作Model,Model通过数据绑定操作View
View: DOM

1）响应式：vue如何监听到data的每个属性的变化

修改data后，vue立刻监听到，因为data属性被代理到vm(this)上

2）模板引擎：vue的模板如何被解析，指令如何处理

1>模板本质是字符串，与html格式很像，但有很大的区别。其本质是有逻辑(V-IF,V-FOR)、可嵌入JS变量字符串，最后还要转换为html来显示。
模板最终必须转换成JS代码，因为——
（1）有逻辑，必须用JS才能实现（图灵完备语言）；
（2）转换为html渲染页面，必须用JS才能实现。
2>因此，模板最终要转化为一个JS函数(render函数)，模板中所有信息都包含在render函数中。

包括首次渲染、修改data后的渲染。这块和虚拟DOM比较相关，打算学习后另开一篇博客写这个~

三、MVVM框架大致流程

（1）模板解析

模板中用到的data中的属性，都变成了JS变量
模板中的v-model v-for v-on 都变成了JS逻辑

（2）响应式开始监听

Object.defineProperty
将data中的属性代理到vm上

（3）首次渲染，显示页面，且绑定依赖

初次渲染，执行render函数，会访问到vm.list,vm.title
会被响应式的get方法监听到
patch将vnode渲染成DOM，初次渲染完成
->data中有很多属性，有些被用到，有些不被用到
被用到的会走到get，不被用到的不会走到get
未走到get中的属性，我们也无需关心set
避免不必要的重复渲染。

（4）data属性变化，触发rerender

修改属性，被响应式的set监听到
set中触发函数，生成新的vnode和prevVnode，进行对比后再渲染到html中。
